## 今回の演習用(演習 3: 完了条件つき修正)

### #1 タスクの期限表示が 9 時間ずれる

- **報告**: 期限を「2026-07-01 09:00(日本時間)」で登録したタスク
  (`task-alice-1`)が、一覧では `2026-07-01 00:00` と表示される。
- **再現手順**: `pnpm dev` で起動し、
  `curl -H "x-api-key: key-alice" "http://localhost:3000/tasks?q=リリース"`
  を実行。`dueDate` が 9 時間前の時刻になっている。
- **期待動作**: 表示用タイムゾーンを指定できるようにする。
  ただし**既存の呼び出し元の挙動(タイムゾーン未指定時)は変えないこと**。
- **ヒント**: 期限の整形は `src/utils/date.ts` の `formatDate` が担当。

### #2 ページネーションで各ページの末尾 1 件が欠ける

- **報告**: Alice はタスクを 8 件持っているのに、
  `GET /tasks?page=1&perPage=3` が 2 件しか返さない。
  `perPage=1` にすると 0 件になる。
- **期待動作**: `page=1&perPage=3` なら 3 件、`perPage=1` なら 1 件返る。
  `total` は影響を受けない(現状も正しい)。

### #3 タスク検索が大文字小文字を区別してしまう

- **報告**: `GET /tasks?q=claude` が 0 件を返す。
  「Claude Code 勉強会の資料レビュー」というタスクがヒットしてほしい。
- **期待動作**: 検索は大文字小文字を区別しない。

## 次回ワークショップ用(並列・マルチエージェント編)

> それぞれ独立した課題です。Worktree で並走させたり、
> 「同じ作業 × N」をサブエージェントに分解する練習に使います。

### #4 機能追加: タスクの CSV エクスポート

`GET /tasks/export.csv` で自分のタスクを CSV でダウンロードできるようにする。

### #5 機能追加: タグ別のタスク集計

`GET /tags/summary` で「タグごとの open / done 件数」を返すエンドポイントを追加する。

### #6 全 POST / PATCH ハンドラに入力バリデーションを追加(同じ作業 × N)

現状、入力バリデーションがあるのは `src/api/tasks.ts` のみ。
`users` / `projects` / `tags` の POST は未検証のボディをそのまま使っている。
tasks と同水準のバリデーションを全ハンドラに追加する。

### #7 エラーメッセージの多言語化(同じ作業 × N)

`src/locales/` には `ja.json` しかなく、API ハンドラのエラーメッセージは
ハードコードされている。`t()`(`src/utils/i18n.ts`)経由に統一した上で、
en / zh / ko など複数ロケールへ展開する。

### #8 セキュリティ監査

このリポジトリには**認可漏れや秘密情報の取り扱い不備が複数仕込まれています**。
セキュリティ観点のカスタムエージェント(またはサブエージェント)を作って監査し、
発見した問題を修正する。
